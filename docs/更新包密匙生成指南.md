# 🔐 JAR 数字签名密钥生成指南（Windows + WSL）

> **适用场景**：
> - 使用 Maven 插件对 Spring Boot JAR 自动签名
> - 服务端验证上传包的真实性与完整性
> - 避免包被篡改或伪造

---

## ✅ 一、环境要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10/11（启用 WSL2）或原生 Ubuntu |
| 工具 | `OpenSSL`（WSL 自带） |
| 权限 | 普通用户即可 |

> 💡 若未安装 WSL，请先在 PowerShell 中执行：
> ```powershell
> wsl --install
> ```
> 重启后即可使用 Ubuntu。

---

## ✅ 二、生成 PKCS#8 格式密钥对（推荐）

> **为什么用 PKCS#8？**
> - 你的 Maven 插件使用 `PKCS8EncodedKeySpec`
> - 兼容 Java 标准库，无需额外依赖
> - 内容以 `-----BEGIN PRIVATE KEY-----` 开头

### 步骤 1：打开 WSL 终端

在 Windows 搜索栏输入 **“Ubuntu”** 并打开。

### 步骤 2：创建安全目录（可选但推荐）

```bash
mkdir -p ~/.secure
cd ~/.secure
```

### 步骤 3：生成 RSA-2048 私钥（PKCS#8 格式）

```bash
openssl genpkey -algorithm RSA -out release-private.pem -pkeyopt rsa_keygen_bits:2048
```

> 🔒 该命令生成**无密码**私钥，适用于自动化构建。  
> 如需密码保护（不推荐用于 CI），加 `-aes256` 参数。

### 步骤 4：导出公钥（PEM 格式）

```bash
openssl pkey -in release-private.pem -pubout -out release-public.pem
```

### 步骤 5：验证密钥格式

```bash
# 检查私钥（应显示 "BEGIN PRIVATE KEY"）
head -n 2 release-private.pem

# 检查公钥（应显示 "BEGIN PUBLIC KEY"）
head -n 2 release-public.pem
```

✅ 正确输出示例：
```
-----BEGIN PRIVATE KEY-----
-----BEGIN PUBLIC KEY-----
```

### 步骤 6：设置文件权限（安全加固）

```bash
chmod 600 release-private.pem    # 仅所有者可读写
chmod 644 release-public.pem     # 所有者读写，组和其他只读
```

## ✅ 三、文件用途说明

| 文件 | 路径（示例） | 用途 | 是否可公开 |
|------|--------------|------|-----------|
| `release-private.pem` | `C:\Users\sweeter\.wsl\.secure\release-private.pem` | **构建时签名**（Maven 插件读取） | ❌ **绝对保密** |
| `release-public.pem` | 随服务端部署（如 `/opt/app/keys/release-public.pem`） | **服务端验签** | ✅ 可公开 |

> 💡 在 Windows 中访问 WSL 文件：  
> WSL 路径 `/home/sweeter/.secure/` 对应 Windows 路径：  
> `\\wsl$\Ubuntu\home\sweeter\.secure\`

---

## ✅ 四、验证密钥对是否有效（可选）

```bash
# 1. 创建测试文件
echo "test" > test.txt

# 2. 用私钥签名
openssl dgst -sha256 -sign release-private.pem -out test.sig test.txt

# 3. 用公钥验证
openssl dgst -sha256 -verify release-public.pem -signature test.sig test.txt
```

✅ 成功输出：
```
Verified OK
```

## ✅ 五、集成到你的项目

### 1. Maven 插件配置（`pom.xml`）

```xml
<plugin>
  <groupId>com.x.edge</groupId>
  <artifactId>maven-jar-sign-plugin</artifactId>
  <version>1.0.0</version>
  <configuration>
    <!-- Windows 路径需转义或使用 WSL 路径 -->
    <privateKeyPath>${user.home}/.secure/release-private.pem</privateKeyPath>
  </configuration>
</plugin>
```

> 💡 在 WSL 中构建时，`${user.home}` 指向 `/home/sweeter`  
> 在 Windows 原生命令行中构建，需使用绝对路径如 `C:\\keys\\release-private.pem`

### 2. 服务端验证

- 将 `release-public.pem` 部署到服务端安全目录
- 使用 `JarSignatureVerifier` 工具类加载公钥并验证 `.bin` 或 `.sig` 文件

---

## 🔒 六、安全最佳实践

1. **私钥绝不提交到 Git**
   ```bash
   echo "*.pem" >> ~/.gitignore
   ```
2. **私钥仅保留在构建机**（如你的开发电脑或 CI 服务器）
3. **公钥部署时设为只读**
   ```bash
   chmod 444 /opt/app/keys/release-public.pem
   ```
4. **定期轮换密钥**（建议每年一次）

---

## 📎 附：密钥格式对比

| 格式 | 私钥头部 | Java 支持方式 |
|------|--------|-------------|
| **PKCS#8（推荐）** | `-----BEGIN PRIVATE KEY-----` | `PKCS8EncodedKeySpec` |
| PKCS#1（传统） | `-----BEGIN RSA PRIVATE KEY-----` | 需 Bouncy Castle 或转换 |

> ❌ 不要使用 `openssl genrsa` 生成的传统格式，除非你额外转换。
