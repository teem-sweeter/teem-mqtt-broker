# 安全策略

## 报告安全漏洞

如果你发现了一个安全漏洞，请**不要**通过公开的 Issue 报告。

请通过以下方式联系我们：

- 发送邮件至项目维护者（请在 README 中查找联系方式）

请在报告中包含：

- 漏洞描述
- 复现步骤
- 可能的影响范围
- 修复建议（如果有）

我们会在 48 小时内确认收到报告，并在 7 天内提供修复计划。

## 安全最佳实践

### 部署安全

- **修改默认凭据**：首次部署后立即修改默认的 MQTT 用户名密码和 JWT Secret
- **启用 TLS**：生产环境必须为 MQTT 连接启用 TLS/SSL 加密
- **限制匿名访问**：生产环境建议关闭 `allow-anonymous`
- **数据库加密**：H2 数据库已内置 AES 加密，请妥善保管数据库密码

### 配置安全

- 通过环境变量设置敏感配置，避免硬编码：
  - `JWT_SECRET` — JWT 签名密钥
  - MQTT 密码
  - H2 数据库密码

### 网络安全

- 不要将 MQTT Broker 端口（1883/8083）暴露到公网，除非必要
- 使用防火墙限制管理端口（8080）的访问
- 生产环境建议使用反向代理（如 Nginx）并启用 HTTPS

## 已知安全限制

- 本项目使用嵌入式 H2 数据库，适合中小规模部署，大规模生产环境建议替换为外部数据库
- 在线升级功能依赖 RSA 签名验证，请确保签名密钥的安全管理
